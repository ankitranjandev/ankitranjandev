      <br><br>
      A Map is essentially a Set of key-value pairs, where each key is unique. The hash table gives you O(1) lookup, insertion, and deletion — by key.
      <br><br>
      <pre><code>// List — must search to find user&#10;var users = [user1, user2, user3];&#10;var found = users.firstWhere((u) => u.id == 'user_123');  // O(n)&#10;&#10;// Map — direct lookup by key&#10;var users = {'user_123': user1, 'user_456': user2};&#10;var found = users['user_123'];  // O(1)</code></pre>

    title: "Map gotchas — null values, missing keys, and empty {}"
    summary: |
      Maps have a few surprises that catch developers off guard.
      <br><br>
      <strong>Gotcha 1: Empty <code>{}</code> creates a Map, not a Set</strong>
      <pre><code>var oops = {};  // Map&lt;dynamic, dynamic&gt;, NOT Set!&#10;var set = &lt;String&gt;{};  // This is a Set&#10;var map = &lt;String, int&gt;{};  // This is a Map</code></pre>
      <strong>Gotcha 2: <code>map[key]</code> returns null for missing keys</strong>
      <pre><code>var map = {'a': 1};&#10;print(map['b']);  // null, not an error!&#10;&#10;// Use containsKey() to distinguish&#10;if (map.containsKey('b')) {&#10;  // Key exists (value might be null)&#10;} else {&#10;  // Key doesn't exist&#10;}</code></pre>
      <strong>Gotcha 3: Null values are valid</strong>
      <pre><code>var map = {'a': null};&#10;print(map['a']);  // null&#10;print(map['b']);  // also null!&#10;&#10;// These are different situations:&#10;print(map.containsKey('a'));  // true&#10;print(map.containsKey('b'));  // false</code></pre>

      <pre><code>var map = {'a': 1, 'b': 2, 'c': 3};&#10;&#10;// BAD — ConcurrentModificationError&#10;for (var key in map.keys) {&#10;  if (map[key] == 2) map.remove(key);  // throws!&#10;}&#10;&#10;// GOOD — collect keys first, then remove&#10;var toRemove = map.keys.where((k) => map[k] == 2).toList();&#10;toRemove.forEach(map.remove);</code></pre>

  - id: performance-considerations
    title: "Performance considerations"
    summary: |
      Maps share the same hash table performance characteristics as Sets.
      <br><br>
      <strong>Operation complexity:</strong>
      <br>
      - <code>map[key]</code> — O(1) average<br>
      - <code>map[key] = value</code> — O(1) amortised<br>
      - <code>map.remove(key)</code> — O(1) average<br>
      - <code>map.containsKey(key)</code> — O(1) average<br>
      - <code>map.containsValue(value)</code> — O(n) (must scan all values!)<br>
      - Iteration — O(n)
      <br><br>
      <strong>Key insight:</strong> <code>containsValue()</code> is O(n) — it can't use hashing because values aren't indexed. If you need fast value lookup, maintain a reverse Map.
      <br><br>
      <pre><code>// Slow: O(n) value lookup&#10;var userById = &lt;String, User&gt;{};&#10;var found = userById.values.firstWhere((u) => u.email == email);&#10;&#10;// Fast: O(1) with reverse index&#10;var userById = &lt;String, User&gt;{};&#10;var userByEmail = &lt;String, User&gt;{};&#10;&#10;void addUser(User u) {&#10;  userById[u.id] = u;&#10;  userByEmail[u.email] = u;  // reverse index&#10;}&#10;&#10;var found = userByEmail[email];  // O(1)</code></pre>

      <strong>When to parallelise Map operations:</strong>
      <br>
      - Large Maps (>10k entries)<br>
      - Expensive per-entry computations<br>
      - Bulk transformations
      <br><br>
      <strong>When NOT to parallelise:</strong>
      <br>
      - Simple lookups (already O(1))<br>
      - Small Maps (overhead exceeds benefit)
