  - id: floating-insight
    title: "Let the point float"
    summary: |
      Here is the idea. Instead of fixing where the decimal point lives, we let it slide.
      <br><br>
      For <code>123.456</code>, the point sits between the 3 and the 4.
      <br>
      For <code>99 999 999</code>, the point slides way to the right.
      <br>
      For <code>0.00000001</code>, the point slides way to the left.
      <br><br>
      In each case, we use all our switches to store the actual digits — none go to waste.
      <br><br>
      To make this work, we mention all the digits together without the decimal, then set aside a few switches to store the position of the point, and one more switch for the sign.
      <br><br>
      That is the basic idea of a <strong>floating-point number</strong>:
      <br><br>
      • some bits for the digits (we call this the <strong>mantissa</strong>)
      <br>
      • some bits for the position of the point (the <strong>exponent</strong>)
      <br>
      • one bit for the sign
      <br><br>
      Most modern languages, including Dart, use a standard called <strong>IEEE 754</strong> to define exactly how those bits are arranged. Let's look inside.

  - id: special-values
    title: "NaN, Infinity, and minus zero"
    summary: "IEEE 754 reserves some bit patterns for special values that aren't ordinary numbers. They exist so that numerical computations can keep running without crashing, even when the maths is undefined.<br><br><pre><code>double inf    = 1.0 / 0.0;   // Infinity\ndouble negInf = -1.0 / 0.0;  // -Infinity\ndouble nan    = 0.0 / 0.0;   // NaN  (Not a Number)\ndouble negZero = -0.0;        // -0</code></pre>A few quirks worth knowing about:<br><br><strong>NaN is not equal to anything — including itself.</strong> This trips everyone up the first time.<br><br><pre><code>double x = 0.0 / 0.0;  // NaN\nprint(x == x);         // false!\nprint(x.isNaN);        // true — use this instead</code></pre>So to check whether a value is NaN, always use <code>.isNaN</code>, never <code>== NaN</code>.<br><br><strong>Positive and negative zero compare equal, but behave differently when used.</strong><br><br><pre><code>print(0.0 == -0.0);    // true\nprint(1 / 0.0);        // Infinity\nprint(1 / -0.0);       // -Infinity</code></pre><strong>Infinity is a real value in Dart.</strong> Dividing by zero does not throw — it gives us <code>Infinity</code> (or <code>-Infinity</code>), and our program keeps running. We can check with <code>.isInfinite</code> or <code>.isFinite</code>.<br><br>This is a feature, not a bug. Many numerical algorithms (especially in physics simulations and signal processing) deliberately let values flow through Infinity and NaN, checking only at the end whether the result is finite."

  - id: typed-floats
    title: "Float32List vs Float64List"
    summary: |
      Just as integers had typed list variants, <code>dart:typed_data</code> gives us packed, unboxed storage for floating-point numbers.
      <br><br>
      • <code>Float64List</code> — 8 bytes per element, full IEEE 754 double precision
      <br>
      • <code>Float32List</code> — 4 bytes per element, single precision (about 7 decimal digits)
      <br><br>
      <code>Float64List</code> matches what we get with plain <code>double</code> — same precision, same range, just packed contiguously in memory without boxing. This is what we want for general numeric work.
      <br><br>
      <code>Float32List</code> trades half the precision for half the memory. That trade is worth it in two big domains:
      <br><br>
      • <strong>Graphics</strong> — vertex positions, colours, texture coordinates. GPUs work in 32-bit floats natively, so passing them <code>Float32List</code> avoids a conversion step.
      <br>
      • <strong>Machine learning</strong> — neural network weights and activations, where speed and memory matter more than the last few digits of precision.
      <br><br>
      For finance, simulation, or anything where rounding errors must be kept tiny, stick with <code>Float64List</code> (or plain doubles).
      <br><br>
      <pre><code>import 'dart:typed_data';

      // Vertex buffer for a 3D model
      Float32List positions = Float32List(vertexCount * 3);

      // Scientific computation
      Float64List samples = Float64List(1000000);</code></pre>
