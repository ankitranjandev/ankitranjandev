  - id: what-is-a-stream
    title: "What is a Stream?"
    summary: |
      A Future delivers one value. But what if you need multiple values over time? That's what Streams are for.
      <br><br>
      Think about user input. A button can be tapped many times. A text field changes with every keystroke. A GPS sensor sends location updates continuously. These aren't single values — they're sequences of events arriving over time.

  - id: single-vs-broadcast
    title: "Single-subscription vs broadcast streams"
    summary: |
      Not all streams behave the same. Dart has two kinds, and mixing them up causes bugs.
      <br><br>
      <strong>Single-subscription streams</strong> can only be listened to once. They're designed for sequences where you don't want to miss any data — like reading a file or receiving an HTTP response body.
      <br><br>
      <pre><code>// Single-subscription: only one listener allowed
      var stream = File('data.txt').openRead();

      stream.listen((chunk) => print('Got \${chunk.length} bytes'));

      // This throws StateError!
      stream.listen((chunk) => print('Another listener'));</code></pre>
      <strong>Broadcast streams</strong> allow multiple listeners. They're for events that happen regardless of who's listening — like button clicks or timer ticks.
      <br><br>
      <pre><code>// Broadcast: multiple listeners allowed
      var controller = StreamController&lt;int&gt;.broadcast();

      controller.stream.listen((n) => print('Listener A: \$n'));
      controller.stream.listen((n) => print('Listener B: \$n'));

      controller.add(1);  // Both listeners receive 1
      controller.add(2);  // Both listeners receive 2</code></pre>

      <strong>Key difference:</strong> Single-subscription streams buffer data until someone listens. Broadcast streams fire events immediately — if no one is listening, the data is lost.
      <br><br>
      <strong>Converting between them.</strong> You can convert a single-subscription stream to broadcast with <code>asBroadcastStream()</code>.
      <br><br>
      <pre><code>var singleStream = countDown(5);
      var broadcast = singleStream.asBroadcastStream();

      broadcast.listen((n) => print('A: \$n'));
      broadcast.listen((n) => print('B: \$n'));  // Now this works</code></pre>

  - id: listening-to-streams
    title: "Listening to streams"
    summary: |
      There are two main ways to consume a stream: <code>listen()</code> for fine-grained control, and <code>await for</code> for simpler code.
      <br><br>
      <strong>The listen() method</strong> gives you full control. It returns a StreamSubscription that you can pause, resume, or cancel.
      <br><br>
      <pre><code>var subscription = stream.listen(
        (data) => print('Data: \$data'),     // onData
        onError: (err) => print('Error: \$err'),
        onDone: () => print('Stream closed'),
        cancelOnError: false,  // keep listening after errors
      );

      // Later...
      subscription.pause();   // stop receiving events
      subscription.resume();  // start again
      subscription.cancel();  // unsubscribe completely</code></pre>
      <strong>The await for loop</strong> is cleaner when you just want to process each value.
      <br><br>
      <pre><code>Future&lt;void&gt; processStream(Stream&lt;int&gt; stream) async {
        await for (var value in stream) {
          print('Got: \$value');
        }
        print('Stream finished');
      }</code></pre>
      <strong>Differences between listen() and await for:</strong>
      <br>• <code>listen()</code> is non-blocking — code after it runs immediately
      <br>• <code>await for</code> blocks until the stream closes
      <br>• <code>listen()</code> gives you a subscription to control
      <br>• <code>await for</code> has simpler syntax but less control
      <br><br>
      <pre><code>// listen() is non-blocking
      stream.listen((v) => print(v));
      print('This prints immediately');

      // await for blocks
      await for (var v in stream) {
        print(v);
      }
      print('This prints after stream closes');</code></pre>
      <strong>Cancellation with await for.</strong> Breaking out of the loop cancels the subscription automatically.
      <br><br>
      <pre><code>await for (var value in stream) {
        if (value > 100) break;  // automatically cancels subscription
        print(value);
      }</code></pre>

    summary: |
      Just like Lists have <code>map()</code> and <code>where()</code>, Streams have transformation methods. Each one returns a new Stream.

      stream.map((x) => x * 2)

      stream.where((x) => x > 10)

      stream.take(5)

      // Skip first N
      stream.skip(3)

      // Remove consecutive duplicates
      stream.distinct()</code></pre>

      <pre><code>// From button tap events to processed data
      buttonTaps
          .where((tap) => tap.button == 'submit')
          .map((tap) => tap.formData)
          .distinct()
          .take(10)
          .listen((data) => submitForm(data));</code></pre>

  - id: stream-controller
    title: "StreamController — creating your own streams"
    summary: |
      <code>StreamController</code> is the bridge between imperative code and streams. You push values in with <code>add()</code>, and listeners receive them through the stream.
      <br><br>
      <pre><code>var controller = StreamController&lt;int&gt;();

      // The stream side — what listeners see
      controller.stream.listen((value) {
        print('Received: \$value');
      });

      // The sink side — where you push values
      controller.add(1);
      controller.add(2);
      controller.add(3);
      controller.close();  // signal no more values</code></pre>
      <strong>Handling errors.</strong> Use <code>addError()</code> to send errors through the stream.
      <br><br>
      <pre><code>controller.add(1);
      controller.addError(Exception('Something went wrong'));
      controller.add(2);  // listeners still receive this</code></pre>
      <strong>Broadcast controllers.</strong> By default, StreamController creates single-subscription streams. For multiple listeners, use the broadcast constructor.
      <br><br>
      <pre><code>var broadcast = StreamController&lt;int&gt;.broadcast();

      broadcast.stream.listen((v) => print('A: \$v'));
      broadcast.stream.listen((v) => print('B: \$v'));

      broadcast.add(42);  // Both receive 42</code></pre>
      <strong>Cleanup is essential.</strong> Always close controllers when done to prevent memory leaks.
      <br><br>
      <pre><code>class CounterService {
        final _controller = StreamController&lt;int&gt;.broadcast();

        Stream&lt;int&gt; get countStream => _controller.stream;

        void increment() => _controller.add(_count++);

        void dispose() {
          _controller.close();  // Important!
        }
      }</code></pre>
      <strong>Back-pressure with onListen/onPause/onResume.</strong> For more control, use the callback parameters.
      <br><br>
      <pre><code>var controller = StreamController&lt;int&gt;(
        onListen: () => print('Someone subscribed'),
        onPause: () => print('Listener paused'),
        onResume: () => print('Listener resumed'),
        onCancel: () => print('Listener cancelled'),
      );</code></pre>

  - id: async-generators
    title: "async* generators and yield"
    summary: |
      We saw <code>sync*</code> generators in the functions episode — they produce synchronous iterables. The async variant, <code>async*</code>, produces Streams.
      <br><br>
      <pre><code>// sync* returns Iterable, uses yield
      Iterable&lt;int&gt; countSync(int n) sync* {
        for (var i = 0; i < n; i++) {
          yield i;
        }
      }

      // async* returns Stream, uses yield
      Stream&lt;int&gt; countAsync(int n) async* {
        for (var i = 0; i < n; i++) {
          await Future.delayed(Duration(seconds: 1));
          yield i;
        }
      }</code></pre>
      <strong>yield* for delegation.</strong> Use <code>yield*</code> to forward all values from another stream.
      <br><br>
      <pre><code>Stream&lt;int&gt; countAndDouble(int n) async* {
        yield* countAsync(n);      // yield all values from countAsync
        yield* countAsync(n).map((x) => x * 2);  // then doubled values
      }</code></pre>

  - id: practical-patterns
    title: "Practical patterns — debounce, throttle, and combining"
    summary: |
      Real applications need more than basic transformations. Let's look at common patterns.
      <br><br>
      <strong>Debounce — wait for pause in events.</strong> Useful for search-as-you-type where you don't want to fire a request on every keystroke.
      <br><br>
      <pre><code>// Using rxdart package
      textFieldStream
          .debounceTime(Duration(milliseconds: 300))
          .listen((query) => search(query));

      // Manual implementation
      Stream&lt;T&gt; debounce&lt;T&gt;(Stream&lt;T&gt; source, Duration duration) async* {
        Timer? timer;
        T? lastValue;
        var hasValue = false;

        await for (var value in source) {
          timer?.cancel();
          lastValue = value;
          hasValue = true;
          timer = Timer(duration, () {});
          await timer.tick;  // simplified
          if (hasValue) {
            yield lastValue as T;
            hasValue = false;
          }
        }
      }</code></pre>
      <strong>Throttle — limit event frequency.</strong> Emit at most one value per time window.
      <br><br>
      <pre><code>// Limit scroll events to 60fps
      scrollEvents
          .throttleTime(Duration(milliseconds: 16))
          .listen((pos) => updateParallax(pos));</code></pre>
      <strong>Combining streams.</strong> Merge multiple sources into one.
      <br><br>
      <pre><code>// Merge: interleave events from multiple streams
      var combined = StreamGroup.merge([stream1, stream2, stream3]);

      // Zip: pair up values (waits for both)
      var zipped = Rx.zip2(namesStream, agesStream,
          (name, age) => '\$name is \$age');

      // CombineLatest: emit when any stream emits
      var latest = Rx.combineLatest2(tempStream, humidityStream,
          (temp, humidity) => 'Temp: \$temp, Humidity: \$humidity');</code></pre>
      <strong>Error recovery.</strong> Handle errors without stopping the stream.
      <br><br>
      <pre><code>stream
          .handleError((error) => print('Caught: \$error'))
          .listen((data) => process(data));

      // Or retry on failure
      stream
          .onErrorRetry(maxRetries: 3)
          .listen((data) => process(data));</code></pre>
      <strong>Disposing properly in Flutter.</strong> Always cancel subscriptions when your widget is disposed.
      <br><br>
      <pre><code>class _MyWidgetState extends State&lt;MyWidget&gt; {
        late StreamSubscription _subscription;

        @override
        void initState() {
          super.initState();
          _subscription = dataStream.listen(_onData);
        }

        @override
        void dispose() {
          _subscription.cancel();  // Prevent memory leaks!
          super.dispose();
        }

        void _onData(data) => setState(() => _data = data);
      }</code></pre>
